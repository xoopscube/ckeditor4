CKEditor4 Overview

Proven, enterprise-grade WYSIWYG HTML editor with wide browser compatibility, including legacy browsers.

This module is the default WYSIWYG editor to create and manage your web content easily.

WYSIWYG implies a user interface that allows the user to view something very similar to the end result while the document is being created.

Main Features

The XOOPSCube module CKEditor4 is released by default with a customized Full package bundle and provides out of the box:

  • Control Panel preferences settings e.g. toolbars, user group permissions, ui color.
  • Localization languages installed by default : English, French, Japanese, Portuguese.
  • Template a Single File Component for frontend and backend.
  • Automatic change of editor HTML or BBCode depending on modules and user group permissions.
  • Automatic switch of ToolBar based on modules preferences and user group permissions.
  • BBCode editor extends the CKEditor standard bbcode plugin
  • HTML editor with custom Toolbar for each user group
  • Extra Plugins customized CodeMirror, oEmbed, Paste (raw text, formatted or code).
  • PHP mode for PHP code blocks without the <?php opening tag.
  • Smarty mode for Smarty Template Engine code blocks.
  • elFinder open-source web file manager with cloud storage settings.

CKEditor4 module Preferences

  • Toolbar for Admin : CKEditor's "config.toolbar" for admin.
  • Toolbar UI Color : Setting Editor UI Color with hexadecimal color code eg. #405060
  • Toolbar for registed user : CKEditor's "onfig.toolbar" for registed user.
  • Toolbar for guest : CKEditor's "config.toolbar" for guest.
  • Toolbar for BBcode plugin : CKEditor's "config.toolbar" for BBcode plugin.
  • CSS URL of editing area : URL of external CSS applied to edit area is set up per line.
    * A special value "<head>" reads automatically CSS specified in <head> in HTML of a display page.
  • Extra plugin : CKEditor's "config.extraPlugins" setting.
  • Custom config URL : CKEditor's "config.customConfig" setting.
  • Enter Mode : CKEditor's "config.enterMode" setting.
  • Shift+Enter Mode : CKEditor's "config.shiftEnterMode" setting.
  • Allow All Content : CKEditor's "config.allowedContent" setting.
  • Auto Paragraph : CKEditor's "config.autoParagraph" setting.
  • X-elFinder : Set X-elFinder directory name for server browser.

Implement CKEditor to textarea.

  1. Clone/Duplicate your Templates to create a new Set.
  2. Edit any installed module's templates with a textarea and add a new parameter editor=html 
<{xoops_dhtmltarea editor=html}>
Note ! Remember to update your new template set.
The CKEditor4 module won't make any data conversion.
Any textarea with previous data will be automatically escaped.

Customize CKEditor4

User Interface color

The CKEditor UI color can be adjusted from the module's preferences Admin > CKEditor > Preferences to match the look and feel of a website or an application. In order to define the color of the user interface, use the config.uiColor configuration setting which accepts an RGB color code.

The editor instance UI color was set by default to the theme RGB value.

Please note that this configuration option can only be used for editor skins that are compatible with the so-called "chameleon" feature, for example:

  • Moono-Lisa
  • Moono
  • Kama
Note :
Refer to the Setting Editor User Interface Color article to learn more about this feature : uicolor ⭧

Customize config.js


CKEDITOR.editorConfig = function( config ) {
	// Define changes to default configuration here. For example:
	// config.language = 'fr';
	// config.uiColor = '#AADC6E';
    config.docType = '';
    // Pressing Enter will create a new 
 element.
    config.enterMode = CKEDITOR.ENTER_P;
    // Pressing Shift+Enter will create a new 
 element.
    config.shiftEnterMode = CKEDITOR.ENTER_BR;
};

CKEDITOR.config

Stores default configuration settings. Changes to this object are reflected in all editor instances, if not specified otherwise for a particular instance.

Read more about setting CKEditor configuration in the documentation.
